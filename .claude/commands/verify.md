# Verifyコマンド

現在のコードベース状態に対して包括的な検証を実行。

## 手順

この正確な順序で検証を実行:

1. **ビルドチェック**
   - このプロジェクトのビルドコマンドを実行
   - 失敗した場合、エラーを報告して停止

2. **型チェック**
   - TypeScript/型チェッカーを実行
   - すべてのエラーをファイル:行で報告

3. **Lintチェック**
   - リンターを実行
   - 警告とエラーを報告

4. **テストスイート**
   - すべてのテストを実行
   - 合格/失敗数を報告
   - カバレッジ率を報告

5. **console.log監査**
   - ソースファイル内のconsole.logを検索
   - 場所を報告

6. **Gitステータス**
   - コミットされていない変更を表示
   - 前回のコミット以降の変更ファイルを表示

## 出力

簡潔な検証レポートを生成:

```
VERIFICATION: [PASS/FAIL]

ビルド:    [OK/FAIL]
型:        [OK/Xエラー]
Lint:      [OK/X問題]
テスト:    [X/Y合格、Z%カバレッジ]
シークレット: [OK/X件発見]
ログ:      [OK/X件console.log]

PR準備完了: [YES/NO]
```

クリティカルな問題がある場合は、修正提案とともにリストアップ。

## 引数

$ARGUMENTSの選択肢:
- `quick` - ビルド + 型のみ
- `full` - すべてのチェック（デフォルト）
- `pre-commit` - コミットに関連するチェック
- `pre-pr` - 完全なチェック + セキュリティスキャン
