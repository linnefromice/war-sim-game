# タスク完了チェックリスト

コードを変更する実装タスクの完了前に適用。

---

## 完了条件

コードを変更した場合、以下の全ステップを完了するまでタスクは未完了とする。

### Step 1: 品質チェック

| 順序 | コマンド     | 目的               |
| ---- | ------------ | ------------------ |
| 1    | `yarn lint`  | ESLint（警告0方針）|
| 2    | `yarn build` | 型チェック + ビルド |
| 3    | `yarn test`  | テスト全通過       |

失敗時はエラーを修正して再実行する。

### Step 2: コミット

- `git status` で変更内容を確認
- `git add` で変更ファイルをステージング
- `git-workflow.md` の Conventional Commits 形式でコミット

### Step 3: プッシュ & PR

main ブランチ以外で作業している場合:

- `git push -u origin <branch>` でリモートに反映
- `gh pr create` でPR作成（未作成の場合）
- PR URLをユーザーに報告

---

## 適用外

以下の場合はこのチェックリストをスキップできる:

- 調査・分析のみのタスク（コード変更なし）
- ユーザーが明示的に「コミットしないで」「チェック不要」と指示した場合
